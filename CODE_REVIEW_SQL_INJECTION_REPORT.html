🔴 Temuan Kerentanan

Issue #1: Potential SQL Injection in whereRaw()

Severity: MEDIUM
File: app/Http/Controllers/Admin/FieldController.php
Lines: 258-260, 268-270

Deskripsi:

Method closeField() menggunakan whereRaw() untuk query timestamp yang kompleks. Meskipun ada parameter binding, penggunaan raw SQL dapat meningkatkan permukaan serangan jika validasi tidak sempurna.

Vulnerable Code:

❌ VULNERABLE
BookingDetail::whereHas('booking', function($query) use ($validatedData) { $query->where('fk_field_id', $validatedData['fk_field_id']); }) ->whereRaw('TIMESTAMP(play_date, start_play_time) < ? && TIMESTAMP(play_date, end_play_time) > ?', [ $validatedData['field_closure_end_time'], $validatedData['field_closure_start_time'], ]) ->where('status', '!=', self::STR_CANCELLED) ->update(['status' => self::STR_FIELD_CLOSURE]);

Potensi Attack Vector:

// Meski di-bind dengan (?), input string berbahaya bisa membebani database POST /api/admin/close-field { "fk_field_id": 1, "field_closure_start_time": "2025-01-01 10:00:00' OR '1'='1", "field_closure_end_time": "2025-01-01 11:00:00" }

Impact:

  • Unauthorized data modification
  • Information disclosure
  • Database denial of service

🛡️ Best Practices untuk Pencegahan SQL Injection

1. ✅ Selalu Gunakan Parameter Binding

❌ JANGAN Lakukan Ini:
$query = "SELECT * FROM users WHERE id = " . $_GET['id']; $result = DB::select($query);
✅ LAKUKAN Ini:
// Menggunakan Laravel Query Builder $user = User::where('id', $request->id)->first(); // Atau dengan raw query dan binding $user = DB::select('SELECT * FROM users WHERE id = ?', [$request->id]);

2. ✅ Gunakan Eloquent ORM (Default Choice)

Eloquent otomatis melakukan parameter binding dan sanitization
// ✅ AMAN - Eloquent ORM $users = User::where('email', $request->email)->get(); $posts = Post::where('title', 'LIKE', "%{$search}%")->get(); $user = User::findOrFail($id);

3. ✅ Implementasi Input Validation yang Ketat

Selalu validate dan type-cast user input:
// ✅ AMAN - Strict Validation $validated = $request->validate([ 'user_id' => 'required|integer|exists:users,id', 'email' => 'required|email|max:255', 'date' => 'required|date_format:Y-m-d|after:now', 'amount' => 'required|numeric|min:0|max:999999', ]); // Data sudah valid dan aman untuk digunakan $user = User::findOrFail($validated['user_id']);

4. ✅ Hindari String Concatenation

Jangan pernah concatenate string untuk SQL queries:
// ❌ SANGAT BERBAHAYA $name = $request->name; $query = "SELECT * FROM users WHERE name = '" . $name . "'"; $result = DB::select($query);
Gunakan method yang aman:
// ✅ AMAN $name = $request->name; $users = User::where('name', $name)->get(); // Atau $users = DB::select('SELECT * FROM users WHERE name = ?', [$name]);

5. ✅ Gunakan Prepared Statements

Prepared statements melindungi dari SQL injection:
// ✅ AMAN - Prepared Statement $bookings = DB::select( 'SELECT * FROM bookings WHERE field_id = ? AND booking_date > ?', [$fieldId, $date] ); // Dengan named parameters $bookings = DB::select( 'SELECT * FROM bookings WHERE field_id = :field_id AND status = :status', [':field_id' => $fieldId, ':status' => 'confirmed'] );

6. ✅ Implementasi Principle of Least Privilege

  • Database users hanya punya permission yang diperlukan
  • API endpoints protected dengan authentication & authorization
  • Field access checks sebelum query execution

7. ✅ Error Handling yang Aman

Jangan expose detail database error kepada user:
try { $user = User::findOrFail($id); } catch (ModelNotFoundException $e) { // ✅ AMAN - Generic error message return response()->json([ 'success' => false, 'message' => 'Data tidak ditemukan' ], 404); }
